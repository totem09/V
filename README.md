Ниже представлена единая сводка по темам управления проектами, жизненного цикла и методологиям внедрения автоматизированных информационных систем (АИС).  
Все вопросы и ответы даны в максимально корректной и сжатой форме, без повторов и с учётом структуры оригинала.

## 1. Формирование целей, задач и групп внедрения

**Вопрос**  
Формирование целей, задач и групп внедрения информационных систем (ИС).

**Ответ**  
1. **Цели внедрения ИС**  
   - Повышение эффективности и производительности.  
   - Снижение издержек.  
   - Улучшение качества обслуживания клиентов.  
   - Поддержка принятия управленческих решений.

2. **Задачи внедрения ИС**  
   - Определение требований к системе.  
   - Разработка или выбор подходящей системы.  
   - Планирование ресурсов и сроков внедрения.  
   - Обучение персонала.  
   - Интеграция ИС в текущие бизнес-процессы.

3. **Группы внедрения**  
   - **Руководство проекта**: стратегическое управление, контроль сроков и бюджета.  
   - **Техническая группа**: установка, настройка, тестирование, техподдержка.  
   - **Бизнес-аналитики**: выявление требований, оценка соответствия ИС потребностям.  
   - **Пользователи**: участие в тестировании, обучение, обратная связь.  
   - **Эксперты по изменениям**: адаптация бизнес-процессов под новую ИС.

## 2. Технологии внедрения информационных систем

**Вопрос**  
Технологии внедрения информационных систем.

**Ответ**  
1. **Big Bang (Мгновенное внедрение)**  
   - Преимущества: минимизация времени, единая интеграция.  
   - Недостатки: высокий риск, требуется идеальная подготовка.

2. **Пошаговое внедрение (Phased Approach)**  
   - Преимущества: снижен риск, возможность корректировок после каждого шага.  
   - Недостатки: дольше по времени, возможны сложности интеграции на каждом этапе.

3. **Параллельное внедрение (Parallel Approach)**  
   - Преимущества: снижение рисков, возможность отката.  
   - Недостатки: высокая стоимость параллельной эксплуатации.

4. **Пилотное внедрение (Pilot Implementation)**  
   - Преимущества: возможность протестировать в реальных условиях, снижение рисков.  
   - Недостатки: ограниченность проверки, возможны проблемы при масштабировании.

5. **Аутсорсинг внедрения**  
   - Преимущества: высокая квалификация исполнителей, экономия времени.  
   - Недостатки: зависимость от подрядчиков, сложность контроля.

## 5. Тестирование программ и систем

**Вопрос**  
Тестирование программ и систем.

**Ответ**  
Тестирование — это процесс проверки ПО для выявления дефектов и оценки качества.

1. **Основные виды тестирования**  
   - Функциональное: проверка соответствия требованиям.  
   - Нефункциональное: производительность, безопасность, удобство.  
   - Модульное: тестирование отдельных компонентов.  
   - Интеграционное: проверка взаимодействия модулей.  
   - Системное: полная проверка всей системы.  
   - Приемочное: оценка соответствия ожиданиям заказчика.  
   - Регрессионное: контроль, что изменения не ломают уже работавший функционал.

2. **Этапы тестирования**  
   - Разработка тест-планов и тест-кейсов.  
   - Выполнение тестов.  
   - Анализ результатов.  
   - Сообщение об ошибках и их устранение.  
   - Повторное тестирование.

## 10. Что такое каскадная модель АИС? Нарисуйте схему модели.

**Вопрос**  
Что такое каскадная модель АИС? Нарисуйте схему модели.

**Ответ**  
**Каскадная (Waterfall) модель** — линейный подход к разработке:

1. Анализ требований  
2. Проектирование  
3. Разработка  
4. Тестирование  
5. Внедрение  
6. Поддержка  

Условная схема:

[Требования] -> [Проектирование] -> [Разработка] -> [Тестирование] -> [Внедрение] -> [Поддержка]

Подходит, когда требования чётко определены. Сложна при внесении изменений на поздних этапах.

## 11. Понятие “живой” документации

**Вопрос**  
Что такое “живая” документация?

**Ответ**  
**“Живая” документация** — это документы, которые:
1. Постоянно обновляются в процессе разработки и эксплуатации.  
2. Содержат актуальные сведения об архитектуре, функционале, требованиях.  
3. Интегрированы в процесс разработки (системы контроля версий, вики).  
4. Используются всеми участниками проекта для синхронизации знаний.

Примеры инструментов: Confluence, Notion, GitHub Wiki.

**Вопрос**  
Что такое итерационная модель АИС? Нарисуйте схему модели.

**Ответ**  
**Итерационная модель** — это разработка ПО циклично, итерациями:

1. Планирование  
2. Проектирование  
3. Разработка  
4. Тестирование  
5. Оценка и дальнейшее улучшение

Каждый цикл создаёт рабочую версию продукта, которую можно улучшать в следующих итерациях.

Упрощённая схема (каждая итерация повторяется):

[Планирование] -> [Проектирование] -> [Разработка] -> [Тестирование] -> [Оценка] -> (Следующая итерация)

## 15. Что такое спиральная модель АИС? Нарисуйте схему модели.

**Вопрос**  
Что такое спиральная модель АИС? Нарисуйте схему модели.

**Ответ**  
**Спиральная модель** сочетает в себе итеративный подход с оценкой рисков. Каждый виток содержит:

1. Планирование  
2. Анализ рисков  
3. Разработка и тестирование  
4. Оценка и планирование следующего цикла

Условная схема (каждый виток «спирали» повторяет эти шаги):
```
      [Планирование] -> [Анализ рисков]
             ↓              ↓
      [Разработка и Тестирование] -> [Оценка и следующий виток]
```

5. **Фаза развертывания (Deploying)**  
   - Установка в рабочей среде, обучение, завершение проекта.

6. **Управление проектом** (Project Management)  
   - Пронизывает все фазы, включает контроль, планирование, управление рисками и ресурсами.

## 45. Распишите методы проектирования АИС

**Вопрос**  
Распишите методы проектирования АИС.

**Ответ**  
1. **Структурное проектирование**  
   - Разбиение системы на модули, детальное описание потоков данных.

2. **Объектно-ориентированное проектирование (ООП)**  
   - Классы и объекты, наследование, полиморфизм, инкапсуляция.

3. **Инкрементное проектирование**  
   - Постепенное добавление функционала.

4. **Прототипирование**  
   - Быстрая разработка прототипов для уточнения требований.

5. **Модульное проектирование**  
   - Разделение на независимые модули с чётко определёнными интерфейсами.

6. **CASE-технологии**  
   - Инструменты для моделирования (UML, BPMN) и автоматической генерации кода.
